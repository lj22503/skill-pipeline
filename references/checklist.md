# 技能质量检查清单

## 📋 新技能创建检查清单

### 发现阶段
- [ ] 已搜索生态，确认无匹配技能
- [ ] 记录搜索关键词和结果

### 设计阶段
- [ ] 目标人群明确（2-50 字符）
- [ ] 核心痛点清晰（5-200 字符）
- [ ] 5 维评分完整（1-5 分）
- [ ] 框架绑定合理
- [ ] 场景设计完整

### 创建阶段
- [ ] SKILL.md 存在且格式正确
- [ ] 元数据完整（name/version/description）
- [ ] description 包含触发词
- [ ] 功能描述清晰
- [ ] 适用场景明确
- [ ] 边界条件说明
- [ ] 常见错误章节
- [ ] 使用示例完整
- [ ] 相关资源链接
- [ ] 故障排查章节

### 安全审查
- [ ] 无危险命令（curl/wget/eval/exec/sudo/rm -rf）
- [ ] 权限范围最小化
- [ ] 模板文件完整
- [ ] references 文件存在
- [ ] scripts 文件有执行权限

### 质量评估
- [ ] 元数据完整（≥20 分）
- [ ] 触发清晰度（≥25 分）
- [ ] 结构完整（≥25 分）
- [ ] 内容质量（≥20 分）
- [ ] 规范性（≥10 分）
- [ ] 总分 ≥75 分

### 上线
- [ ] 技能地图已更新
- [ ] 上线日志已记录
- [ ] 用户已确认

## 📋 技能迭代检查清单

### 观察阶段
- [ ] 选择 3-5 个测试 prompt
- [ ] 包含 happy path
- [ ] 包含边缘场景
- [ ] 包含反向测试
- [ ] 记录观察笔记

### 总结阶段
- [ ] 提炼错误模式（3 次规则）
- [ ] 标注根因
- [ ] 评估影响面
- [ ] 排定优先级

### 改进阶段
- [ ] 遵循 JIT 原则（每轮只改一件事）
- [ ] 解释 why（不是堆 MUST/NEVER）
- [ ] 保持信息密度
- [ ] 可验证改动

### 验证阶段
- [ ] 用同一批 prompt 重跑
- [ ] 对比前后结果
- [ ] 判断是否收敛
- [ ] 更新演进日志

## 📋 定期审计检查清单

### 扫描阶段
- [ ] 技能库完整扫描
- [ ] 记录技能数量变化
- [ ] 标记新增/删除技能

### 审查阶段
- [ ] 批量质量评估

- [ ] 标记过期技能

### 报告阶段

## 🚨 一票否决项

以下情况直接不通过：

1. **安全检查失败**
   - 发现恶意代码
   - 权限范围过大
   - 模板文件缺失

2. **质量评估 <60 分**
   - 元数据严重缺失
   - 触发描述不清
   - 结构严重不完整

3. **演进不收敛（3 轮以上）**
   - 职责太宽
   - 测试 prompt 矛盾
   - 底层能力限制
